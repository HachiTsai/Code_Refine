# 規格書: DCS 數位資產元數據 (Dataview) 標準 (v1.0)

## 1. 核心定義 (Core Identity)
本規範定義了 DCS 數位孿生筆記（Obsidian Markdown）中 `%% DCS_FACTS %%` 區塊的結構與語義。所有元數據必須確保與 Core JSON 具備 100% 的雙向同步性。

## 2. Dataview 欄位定義

| 欄位 | 必填 | 格式規範 | 物理意義 |
| :--- | :---: | :--- | :--- |
| **Summary** | ✅ | `[專業描述文本]` | 專家的精煉見解，描述該迴路的核心物理意圖。 |
| **Importance** | ✅ | `[類別] 具體功能價值說明` | 定義該迴路在全廠中的關鍵度與角色。 |
| **Process** | ✅ | `#標籤1 #標籤2` | 屬於哪類生產工藝（如 #壓力控制, #反應監控）。 |
| **Interlock** | ✅ | `[信號或迴路 ID]` | 物理連鎖或邏輯相依性。**嚴禁使用 [[...]] 連結**。 |
| **Signals** | ✅ | `[TAG1], [TAG2]` | 該迴路涉及的所有關鍵實體與邏輯標籤。 |

## 3. Importance 等級與類別規範

為了確保全廠資產的可檢索性，`Importance` 必須遵循 `[類別] 描述` 格式：

- **[核心]**: 涉及主要反應釜 (Reactor) 的生產生命週期、SEQ 序列總指揮。
- **[關鍵控制]**: 涉及安全、產量或品質平衡的關鍵 PID 迴路。
- **[關鍵監視]**: 涉及劇毒、易燃或高壓環境的安全監測點。
- **[監視]**: 一般性的流量、液位或溫度顯示，用於日常庫存或數據參考。
- **[輔助]**: 參數管理 (DATA)、步序切換中轉站或非實體控制邏輯。

## 4. 語義禁令 (Negative Constraints)

1. **禁止雙向連結**: 在 `DCS_FACTS` 區塊的正文中，嚴禁使用 `[[Link]]`。引用標籤時統一使用 `**[TAG]**` 或直接寫入標籤 ID。
2. **禁止廢話**: 摘要與描述應直接切入物理本質，例如「防止乾燒」優於「這是一個保護迴路」。
3. **禁止 Generated 標註**: 嚴禁在筆記中出現「Generated by AI」或類似字眼，所有產出視為正式工程資產。

## 5. 同步協議 (SSOT Handshake)

1. 任何對 Markdown 中 `Summary::` 或 `Importance::` 的修改，必須由 `note-manager` 執行 `update-summary` 動作回寫至 `_assets/30_Digital_Twin/core/` 下的 JSON。
2. JSON 中的 `expert_insight` 對應 Markdown 的 `Summary::`。
3. JSON 中的 `importance` 對應 Markdown 的 `Importance::`。

## 6. DCS 知識資產煉製 SOP (WorkFlow)

本章節定義了使用 `note-manager` 進行資產煉製的標準化作業順序，Agent 必須嚴格遵守：

1. **第一階段：骨架生成 (Skeleton)**
    - **動作**：執行 `--action create-skeleton`。
    - **目標**：建立合規檔名、YAML Frontmatter 與標準 `DCS_FACTS` 區塊。
2. **第二階段：深度收割 (Deep Harvest)**
    - **動作**：執行 `--action auto-refine`。
    - **目標**：自動搜尋全廠 Core JSON 關聯與向量庫證據，產出 **Universal Context**。
3. **第三階段：高品質合成 (Atomic Synthesis)**
    - **動作**：Agent 參考**對應類別的解析規範**（如 `spec-analyze-ddc.md`, `spec-analyze-seq.md` 或 `spec-analyze-bls.md`），並**整合最終摘要**，合成一份包含「元數據 + 深度解析」的完整 Markdown 內容。
    - **核心準則 (Atomic Write)**：嚴禁先注入正文再執行 `update-summary`。必須透過一次性 `write_file` 寫入完全體，以防止標籤同步腳本因快取問題覆蓋掉高品質正文。
4. **第四階段：SSOT 雙向握手 (Handshake)**
    - **動作**：執行 `--action pull-summary`。
    - **目標**：將 Markdown 中已定稿的高品質摘要反向回灌至 Core JSON，完成數據靈魂的雙向對齊。
